# AGENTS.md

## 프로젝트 개요

Vibe Dating은 사용자의 데이트 성향과 상황에 맞는 장소 및 코스 후보를 지도 기반으로 추천하고, 자연어 요청으로 결과를 조정하는 데이트 큐레이션 서비스다.

이 저장소는 현재 앱 구현보다 기획, 도메인 정리, 기술 검토 문서가 중심이다. `AGENTS.md`는 상세 명세가 아니라 새 세션에서 프로젝트 맥락과 문서 위치를 빠르게 파악하기 위한 안내 문서로 유지한다.

## 주요 문서

- `docs/MVP-기능-명세서.md`: MVP 기능 범위와 사용자 경험 기준
- `docs/MVP-기술-검토.md`: 추천, AI 분석, 데이터 수집, 지도 연동 등 구현 관점의 검토
- `docs/사용자-성향.md`: 사용자 성향 및 선호 항목 기준
- `docs/장소-카테고리.md`: 장소 카테고리 분류 기준
- `docs/MVP-기능-명세서-단계/`: MVP 기능 명세를 만들기 위한 단계별 논의 기록
- `docs/api/`: 외부 API, 지도, 플레이스 데이터 관련 조사 자료

새 세션에서 프로젝트를 파악할 때는 다음 순서로 읽는다.

1. `AGENTS.md`
2. `docs/MVP-기능-명세서.md`
3. `docs/MVP-기술-검토.md`
4. 필요한 경우 `docs/사용자-성향.md`, `docs/장소-카테고리.md`
5. 논의 배경이나 세부 결정 흐름이 필요하면 `docs/MVP-기능-명세서-단계/`
6. 외부 API나 데이터 수집 판단이 필요하면 `docs/api/`

## 문서 작성 가이드

- 기능 범위, 사용자 경험, 화면 요구사항은 `docs/MVP-기능-명세서.md`에 정리한다.
- 추천 로직, AI 분석, 크롤링/검색, 지도 연동, 에이전트 처리 같은 구현 검토는 `docs/MVP-기술-검토.md`에 정리한다.
- 사용자 성향 선택지나 선호 항목은 `docs/사용자-성향.md`에 정리한다.
- 장소 카테고리 체계는 `docs/장소-카테고리.md`에 정리한다.
- API 조사나 외부 데이터 출처별 제약은 `docs/api/` 하위 문서에 정리한다.
- 결정 배경을 남겨야 할 때는 `docs/MVP-기능-명세서-단계/`에 있는 단계별 문서를 참고하거나 보완한다.

## 작업 가이드

- 프로젝트 관련 질문에는 먼저 `docs/` 문서를 기준으로 답한다.
- 문서에 없는 내용은 확정된 사실처럼 말하지 않고, 가설이나 제안으로 구분한다.
- 새로운 제품/도메인 결정이 생기면 관련 문서에 반영하는 것을 우선 고려한다.
- 상세 기능, 스키마, 정책, 수치, 검증 지표는 `AGENTS.md`에 넣지 않고 담당 문서에만 둔다.
- 문서 간 역할이 겹칠 때는 사용자 경험은 기능 명세서, 구현 판단은 기술 검토, 분류 기준은 별도 기준 문서로 분리한다.

## 세션 마무리 트리거

사용자가 `"세션 마무리"`라고 말하면, 이번 대화에서 장기적이고 글로벌한 지침으로 남길 만한 후보만 정리해서 사용자에게 보낸다.

- 형식: 키워드와 핵심 중심의 간결한 개괄식
- 포함: 반복 가능한 작업 방식, 장기 선호, 프로젝트 밖에서도 유효한 지침
- 제외: 일회성 작업 내용, 임시 맥락, 특정 버그의 세부 해결 과정
- 저장: 승인 전에는 지침에 반영하지 않고, 사용자가 승인한 항목만 따로 저장 대상으로 다룬다.
