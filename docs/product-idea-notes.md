# Vibe Dating Product Idea Notes

## 해결하려는 문제

- 현재 데이트 코스를 찾으려면 네이버 지도와 웹 검색을 오가며 직접 발품을 팔아야 한다.
- 검색 결과에는 노이즈가 많고, 커플의 취향에 맞는 장소만 빠르게 고르기 어렵다.
- Vibe Dating은 커플이 직접 여러 곳을 찾아보고 비교하는 수고를 줄여주는 데이트 장소 추천 플랫폼을 목표로 한다.

## 핵심 가치

- 네이버 지도, 블로그, 검색 결과 전체를 뒤지는 대신 커플 취향에 맞는 후보를 압축해서 보여준다.
- 장소를 단순 나열하는 앱이 아니라 커플 성향 기반의 데이트 장소 큐레이션 레이어를 만든다.
- 추천된 장소에 대한 호/불호 데이터를 누적해 점점 더 정밀한 추천을 제공한다.

## 타깃

- 장기적으로는 모든 커플을 대상으로 한다.
- 사용 시점은 특정 상황에 제한하지 않고, 위치 기반으로 언제든지 사용할 수 있게 한다.
- MVP에서는 추천 품질을 검증하기 위해 서울 내 일부 핫플 권역부터 시작한다.

## 장소 범위

- 장기적으로는 데이트로 갈 수 있는 모든 장소를 포함한다.
- 예시:
  - 음식점
  - 카페
  - 술집
  - 전시
  - 팝업
  - 산책/공원
  - 공방/체험
  - 쇼핑
  - 액티비티

## 핵심 기능

- 커플 성향 기반 데이트 장소 추천
- 추천된 데이트 장소에 대한 호/불호 표시
- 호/불호 누적 데이터를 기반으로 추천 정밀도 개선
- 지도 위에서 추천된 데이트 장소와 후보 장소 확인
- 향후 지도 위에서 직접 Vibe Dating Course를 편집하고 구성하는 기능

## MVP 방향

- 초반 커플 성향은 태그 선택 방식으로 수집한다.
- 우선 서울에만 적용한다.
- 서울 안에서도 핫플 권역부터 시작한다.
- MVP에서는 추천 장소를 모두 보여주는 것에 집중한다.
- 코스 자동 생성이나 지도 기반 코스 편집은 후순위로 둔다.

## 초기 추천 플로우

1. 사용자가 지역 또는 현재 위치를 기준으로 탐색을 시작한다.
2. 커플의 오늘 무드 또는 성향 태그를 선택한다.
3. 성향 기반으로 추천 장소 목록을 보여준다.
4. 추천 장소를 지도 위 핀과 리스트로 함께 확인한다.
5. 각 장소에 대해 호/불호를 표시한다.
6. 누적된 피드백을 다음 추천에 반영한다.

## 데이터 수집 전략 초안

- 지도 UI는 네이버 지도 API를 사용한다.
- 사용자는 네이버 지도 기반 화면에서 추천 장소와 후보 장소를 확인한다.
- 장소 후보는 네이버 지도 검색 결과를 참고하고, web search, 크롤링, 공개 데이터, 수동 큐레이션 등 다양한 수단으로 보강한다.
- 네이버 지도 검색 결과나 외부 웹 콘텐츠를 그대로 우리 DB의 원본처럼 복제하지 않고, 후보 발견과 검수의 참고 자료로 활용한다.
- 우리 Place DB에는 자체적으로 정리한 장소 정보, 데이트 태그, 추천 점수, 검수 상태, 사용자 호/불호 데이터를 저장한다.
- 장소 카테고리는 내부 기준으로 정규화한다.
- 데이트 감성, 분위기, 실제 적합도는 외부 API 데이터만으로 판단하기 어려우므로 자체 scoring과 수동 검수를 함께 사용한다.
- 초기에는 핵심 권역별로 일부 수동 큐레이션을 섞어 추천 품질을 확보한다.

## 자체 Place DB 방향

- 핵심 자산은 외부 지도/검색 데이터가 아니라 Vibe Dating이 직접 만든 데이트 맥락 데이터다.
- 저장할 핵심 데이터:
  - 장소명
  - 주소
  - 좌표
  - 내부 카테고리
  - 공식 홈페이지/인스타/예약 링크
  - 자체 작성 장소 설명
  - 데이트 적합 태그
  - 추천 이유
  - 검수 상태
  - 출처 URL
  - 호/불호 데이터
  - 추천 점수
- 저장을 피해야 할 데이터:
  - 외부 리뷰 원문
  - 블로그/웹 문장 원문
  - 외부 플랫폼 사진
  - 외부 플랫폼 메뉴 이미지
  - 외부 플랫폼의 검색 순위 자체
  - 외부 플랫폼 평점/리뷰 수의 무분별한 장기 저장
- web search와 크롤링은 후보 발굴, 교차 검증, 최신성 확인, 공식 링크 탐색에 우선 사용한다.
- 외부 콘텐츠를 사용할 때는 출처, 이용약관, robots.txt, 저작권, 저장 가능 범위를 확인해야 한다.

## 필터링 및 스코어링 방향

- 처음에는 평점 2.0 이하 같은 단순 필터링을 고려했으나, "퀄리티 높음/낮음"을 고정 기준으로 나누는 것은 적절하지 않을 수 있다.
- 장소의 품질을 선제적으로 제거하기보다, 추천 스코어링에서 후순위로 밀리게 하는 방식이 더 자연스럽다.
- 실제 데이터 확인 후 구체적인 스코어링 조건을 선정해야 한다.

## 추천 스코어링 v1 아이디어

- 커플 성향 태그 매칭
- 현재 위치 또는 선택 지역과의 거리
- 장소 카테고리 선호도
- 여러 출처에서 반복 발견되는지 여부
- 공식 페이지/예약 링크 존재 여부
- 최근 웹 언급 여부
- 수동 검수 점수
- 데이트 적합 태그
- 최근성 또는 핫함
- 너무 흔한 장소에 대한 패널티
- 기존 호/불호 데이터 기반 보정
- 폐업/휴업 의심 패널티
- 정보 부족 패널티

## 성향 태그 방향

- 성향은 개인 단위가 아니라 커플 단위로 수집한다.
- 초기에는 다양한 태그를 제공한다.
- 예시 태그:
  - 조용한
  - 감성적인
  - 트렌디한
  - 고급스러운
  - 캐주얼한
  - 대화 중심
  - 걷기 좋은
  - 체험형
  - 구경하는
  - 오래 머무는
  - 가성비
  - 기념일급
  - 첫 데이트
  - 오래된 커플
  - 비 오는 날
  - 저녁 데이트
  - 맛집
  - 카페
  - 술
  - 전시
  - 자연
  - 쇼핑
  - 사진
  - 이색 체험
  - 실내 선호
  - 야외 선호
  - 웨이팅 싫음
  - 이동 적게

## 사용자 피드백

- MVP에서는 사용성을 위해 장소별 호/불호만 받는다.
- 추천 이유별 피드백이나 세부 불만 사유 수집은 후순위로 둔다.

## 지도 및 코스 기능

- MVP에서는 네이버 지도 위에 추천 장소와 후보 장소를 리스트/마커로 보여주는 수준으로 시작한다.
- 지도는 네이버 지도 API 기반으로 구현한다.
- 지도는 장소 탐색과 시각화의 인터페이스로 사용하고, 추천 판단은 자체 Place DB와 scoring을 기반으로 한다.
- 추천된 장소를 기반으로 지도 위에서 코스를 커스텀하게 편집하는 전략은 추후 정리한다.
- 경로 생성, 동선 최적화, 코스 자동 추천은 후속 기능으로 고려한다.

## 수익 모델

- 초기 수익 모델은 광고를 우선 고려한다.
- 향후 장소 제휴, 예약 수수료, 프로모션, 프리미엄 추천 등의 가능성도 검토할 수 있다.

## 핵심 가설

- 커플은 검색 결과 전체가 아니라, 자기들 취향에 맞는 소수의 좋은 후보를 원한다.
- Vibe Dating이 줄여주는 핵심 수고는 "어디가 좋고 나쁜지 직접 찾아보고 비교하는 과정"이다.
