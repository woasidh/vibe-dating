# 5. 핵심 사용자 플로우

작성일: 2026-06-14

## 질문

- 사용자가 성향을 입력하는 방식은 무엇이 좋은가?
- 첫 추천 결과는 몇 개 정도 보여주는가?
- 추천 결과는 장소 카드 리스트와 지도 마커 형태가 맞는가?
- 장소 상세는 우리 서비스 안에서 어느 정도 보여줄 것인가?
- 코스 생성은 추천 결과 화면 안에서 바로 요청하는 방식이면 되는가?

## 사용자 답변

- 성향 입력은 태그 선택 방식으로 한다.
- 지도 기반 검색된 플레이스들은 모두 보여준다.
- 사용자 성향 기반 점수 순서대로 보여준다.
- 점수가 높은 곳은 진하고, 낮아질수록 옅은 색으로 표현하는 UX를 생각 중이다.
- 추천 결과는 장소 카드 리스트와 지도 마커 형태가 맞다.
- 장소 카드에는 추천 이유, 간단 요약, 외부 링크 및 평점을 보여준다.
- 코스 생성과 수정 요청은 채팅창에서 바로 입력했으면 좋겠다.

## 정리된 결정

핵심 플로우는 다음과 같다.

1. 사용자가 앱에 진입한다.
2. 회원가입 또는 로그인을 한다.
3. 회원가입 시 기본 데이트 성향 문장을 선택한다.
4. 사용자가 지도 화면에 진입한다.
5. 현재 지도에 보이는 영역을 기준으로 데이트 가능 플레이스가 조회된다.
6. 저장된 사용자 성향을 기반으로 플레이스별 추천 점수가 계산된다.
7. 플레이스는 추천 점수 순서로 리스트에 표시된다.
8. 지도 위 마커는 추천 점수에 따라 시각적 강약이 다르게 표시된다.
9. 사용자는 지도와 리스트를 함께 보며 추천 후보를 확인한다.
10. 각 플레이스 카드에서 간단 요약, 추천 이유, 평점, 외부 링크를 확인한다.
11. 사용자는 채팅창에서 바로 자연어 요청을 입력한다.
12. 서비스는 요청에 따라 추천 결과를 수정하거나 코스를 생성한다.
13. 생성된 코스 또는 수정된 추천 결과가 지도와 리스트에 다시 표시된다.
14. 사용자는 최종적으로 마음에 드는 플레이스의 외부 지도 링크로 이동해 상세 정보를 확인한다.
