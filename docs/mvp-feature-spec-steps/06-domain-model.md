# 6. 도메인 모델

작성일: 2026-06-14

## 질문

- 성향 태그는 어떤 기준으로 구성하고 싶은가?
- 플레이스 점수는 어떤 요소로 계산하면 좋은가?
- 추천 이유는 어떤 식으로 보여주고 싶은가?
- 코스는 어떤 구조로 볼 것인가?
- 바이브 수정에서 사용자가 바꿀 수 있는 대상은 어디까지인가?

## 사용자 답변

- 성향은 문장형으로 선택하게 할 것이다.
- 예시는 다음과 같다.
  - 조용한 게 좋아요.
  - 사진 찍는 걸 좋아해요.
- 우선 데이트 성향에 필요한 문장 몇 개만 넣고 추후 다듬자.
- 플레이스에 대한 정보는 웹 크롤링이든 웹서치든 가져올 것이다.
- 이 정보를 가지고 AI가 각각 태그당 점수를 매긴다.
- 최종적으로는 플레이스별로 태그당 최종 점수가 나온다.
- 이 점수 기반으로 사용자의 성향 태그에 맞는 추천 점수를 낸다.
- 이런 핵심 로직은 별도의 기술 검토 문서에서 다루는 것이 좋겠다.
- 추천 이유도 추천 알고리즘 결과 기반으로 보여주되, 추후 정리한다.
- 코스는 사용자 입력으로 받기 때문에 기본값으로 제시하고 마음에 안 들면 사용자가 수정하도록 한다.
- 모호한 요청은 사용자에게 질문하는 방식도 가능하다.
- 기본 유닛 단위인 장소 추가, 삭제, 수정이 가능하다.
- 코스 수정도 가능하다.
- 수정 요청은 자연어 기반으로 어떤 방식이든 가능하다.
- AI 에이전트를 통해 처리할 것이다.
- 예시는 다음과 같다.
  - 도보 거리 줄이는 편으로 반영해줘.
  - 점심 후 카페를 카페 후 저녁 구조로 바꿔줘.
  - 조금 더 조용한 느낌으로 바꿔줘.

## 정리된 결정

성향은 문장형 선택지로 제공한다.

플레이스는 태그별 점수를 가진다. 사용자 성향 기반 추천 점수는 사용자 성향과 플레이스별 태그 점수를 조합해 만든다.

추천 알고리즘, 크롤링, AI 점수화, 추천 이유 생성, 바이브 수정 에이전트 로직은 별도 기술 검토 문서에서 다룬다.

코스는 여러 플레이스의 조합이며, 사용자가 요청했을 때 생성한다.

바이브 수정은 자연어 기반 편집 요청이며, 장소와 코스를 모두 수정할 수 있다.

수정 가능한 대상은 다음과 같다.

- 장소 추가
- 장소 삭제
- 장소 교체
- 코스 순서 변경
- 코스 구조 변경
- 분위기 변경
- 이동 거리 줄이기
- 카테고리 변경
