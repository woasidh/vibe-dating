# 4. MVP 목표와 제외 범위

작성일: 2026-06-14

## 질문

- MVP에서 반드시 검증하고 싶은 것은 무엇인가?
- MVP에서 데이트 코스는 꼭 자동으로 만들어줘야 하는가?
- 바이브 수정은 MVP에 포함할 것인가?
- MVP에서 명확히 안 할 기능은 무엇인가?

## 사용자 답변

- 성향 기반 장소 추출이 잘 되는지 검증하고 싶다.
- 추천이 실제로 유용한지 검증하고 싶다.
- 대화형 수정 기능이 좋은 사용자 경험을 이끌어내는지 검증하고 싶다.
- 처음에는 장소들만 추천해주고, 코스는 대화형 요청으로부터 생성되는 것으로 하자.
- 사용자가 요청해야만 코스가 만들어지는 방식으로 한다.
- 사용자가 선택한 장소들을 이어주는 것도 가능하다.
- 추천된 플레이스들을 조합해서 경로 수정은 무엇이든 가능한 기능이어야 한다.
- 예시는 다음과 같다.
  - 여기에서 제일 괜찮은 코스 하나만 만들어줘.
  - 두 번째 카페는 마음에 안 들어. 다른 걸로 넣어줘.
  - 저녁 파스타 말고 고기.
- 예약, 결제, 커플 계정 등 핵심 추천/수정 경험과 직접 관련 없는 기능은 제외한다.

## 정리된 결정

MVP에서 검증할 핵심은 다음 세 가지다.

- 성향 기반 장소 추출이 잘 되는가
- 추천이 실제로 유용한가
- 대화형 수정이 좋은 사용자 경험을 만드는가

MVP의 기본 출발점은 장소 추천이다. 코스는 자동으로 항상 생성하지 않고, 사용자가 요청했을 때 생성한다.

바이브 수정은 MVP에 포함한다. 사용자가 추천된 플레이스나 생성된 코스를 대화형으로 수정할 수 있어야 한다.

MVP 제외 범위는 다음과 같다.

- 예약
- 결제
- 커플 계정
- 복잡한 프로필 설문
- 상세 리뷰 작성
- 업주용 장소 등록
- 운영자 검수 어드민 고도화
- 자동 상시 코스 생성
