# 9. 화면 단위 요구사항

작성일: 2026-06-14

## 질문

- 앱은 모바일 앱 중심인가, 데스크톱 웹도 중요한가?
- 지도 기반 추천 화면은 어떤 구조가 좋은가?
- 채팅창은 어디에 있는 게 좋은가?
- 회원가입 시 성향 선택은 몇 개 정도 고르게 할 것인가?
- 지역 선택은 어떤 방식으로 할 것인가?

## 사용자 답변

- 모바일 앱 중심이다.
- 지도 기반 추천 화면은 지도 중심 + 아래/옆에 리스트 구조가 좋다.
- 채팅창 위치는 추천을 원한다.
- 성향 선택은 제한 없이 여러 개 선택할 수 있게 한다.
- 지역 선택은 현재 지도에 보이는 화면을 기반으로 한다.

## 정리된 결정

MVP는 모바일 앱 중심으로 설계한다.

메인 화면은 지도 중심 + 하단 리스트 구조로 한다.

현재 지도에 보이는 화면 영역을 기준으로 플레이스를 추천한다.

채팅창은 지도 화면 하단 고정 입력창 + 확장형 채팅 바텀시트로 설계한다.

성향 문장은 제한 없이 여러 개 선택 가능하다.

## 필요 화면

- 회원가입 화면
- 로그인 화면
- 성향 선택 온보딩 화면
- 지도 기반 추천 화면
- 플레이스 리스트 / 카드 영역
- 지도 마커
- 바이브 수정 채팅창
- 코스 결과 표시 영역
- 내 성향 수정 화면

## 화면별 방향

### 회원가입 화면

- 이메일, 소셜 로그인 등 최소 정보로 가입한다.
- 복잡한 프로필 입력은 제외한다.

### 로그인 화면

- 기존 사용자가 저장된 성향으로 바로 추천받기 위한 진입점이다.

### 성향 선택 온보딩 화면

- 문장형 성향 선택지를 제공한다.
- 사용자는 제한 없이 여러 개 선택할 수 있다.
- 선택한 성향은 기본 데이트 성향으로 저장된다.

### 지도 기반 추천 화면

- 앱의 메인 화면이다.
- 지도 중심 + 하단 리스트 구조로 구성한다.
- 현재 지도에 보이는 화면 영역을 기준으로 플레이스 추천이 이루어진다.
- 사용자가 지도를 이동하거나 확대/축소하면 해당 지도 영역 기반으로 추천 결과를 갱신한다.

### 바이브 수정 채팅창

- 지도 화면 하단에 고정 입력창을 둔다.
- 기본 상태에서는 한 줄 입력창만 보인다.
- 대화가 시작되면 확장형 채팅 바텀시트로 표시한다.
