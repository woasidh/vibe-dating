# 10. 데이터 및 정책 요구사항

작성일: 2026-06-14

## 질문

- 플레이스 정보는 MVP에서 어디까지 저장할 것인가?
- 평점은 네이버, 카카오, Google 중 어디 기준으로 보여줄 것인가?
- 외부 링크는 어떤 것을 우선으로 둘 것인가?
- 웹 크롤링/웹서치로 수집한 정보 중 DB에 저장하면 안 된다고 생각하는 것은 무엇인가?

## 사용자 답변

- 저장할 데이터는 장소명, 주소, 좌표, 카테고리, 외부 링크, 평점, 요약, 태그별 점수다.
- 평점은 여러 출처를 다 사용한다.
- 외부 링크는 네이버가 1순위다.
- 리뷰와 블로그 문장 원문은 저장하지 않는다.
- 사진도 저장하지 않는다.
- 메뉴판 이미지도 저장하지 않는다.
- 평점은 계속 보여줘야 하므로 저장 또는 표시 대상에 포함한다.

## 정리된 결정

## 저장할 플레이스 데이터

- 장소명
- 주소
- 좌표
- 카테고리
- 외부 링크
- 평점
- 간단 요약
- 태그별 점수

## 평점

네이버, 카카오, Google 등 여러 출처의 평점을 함께 사용할 수 있다.

평점은 사용자가 추천 결과를 판단하는 데 필요한 정보이므로 MVP에서 계속 보여준다.

## 외부 링크

외부 링크는 네이버 지도를 1순위로 둔다.

필요에 따라 카카오맵, Google Maps, 공식 홈페이지, 인스타그램 링크를 함께 제공할 수 있다.

## 저장하지 않을 데이터

- 리뷰 원문
- 블로그 문장 원문
- 외부 플랫폼 사진
- 메뉴판 이미지

Vibe Dating은 외부 콘텐츠 원문을 복제하는 서비스가 아니라, 외부 정보와 자체 분석을 기반으로 데이트 맥락에 맞는 추천을 제공하는 서비스로 정의한다.

## 별도 기술 검토 문서 필요 항목

- 웹 크롤링 / 웹서치 기반 플레이스 정보 수집 방식
- 외부 API 사용 범위
- 외부 데이터 저장 가능 범위
- AI 기반 태그별 점수 산정 로직
- 사용자 성향 기반 추천 점수 계산 방식
- 추천 이유 생성 방식
- AI 에이전트 기반 바이브 수정 처리 방식
